# List Comprehension (списковые включения) в Python — полное руководство

## 0. Важное уточнение по терминологии

В русскоязычной среде List Comprehension часто называют **«генератор списка»** или **«списковое включение»** — это одно и то же понятие. Но здесь есть ловушка: в самом Python есть отдельное понятие **generator** (генератор) — это другая, хоть и похожая по синтаксису конструкция (раздел 8). Чтобы не путаться:

- **List comprehension** = `[x for x in ...]` — квадратные скобки, сразу создаёт список в памяти.
- **Generator expression** (генератор) = `(x for x in ...)` — круглые скобки, ничего не создаёт сразу, а лениво выдаёт значения по одному.

Так что «генератор списка» как разговорный термин — это именно list comprehension, а не generator в строгом смысле Python. Дальше в файле я буду называть это «списковое включение», а генераторам посвящён отдельный раздел для сравнения.

## 1. Что это такое

List comprehension — это выражение, которое создаёт новый список на основе существующего итерируемого объекта (списка, строки, range, словаря и т.д.), применяя к каждому элементу некоторое правило и/или условие фильтрации. Это синтаксический сахар над обычным циклом `for` с `.append()`.

```python
# Обычный цикл
result = []
for x in range(5):
    result.append(x ** 2)

# List comprehension
result = [x ** 2 for x in range(5)]
```

Оба варианта дают `[0, 1, 4, 9, 16]`, но второй короче и, как правило, чуть быстрее.

## 2. Базовая структура

Полный синтаксис списочного включения:

```python
[выражение for переменная in итерируемый_объект if условие]
```

Разберём по частям:

| Часть | Обязательна? | Назначение |
|---|---|---|
| `выражение` | да | что именно кладём в новый список (может использовать переменную и любые функции) |
| `for переменная in итерируемый_объект` | да | цикл по источнику данных |
| `if условие` | нет | фильтр — элемент включается, только если условие истинно |

### 2.1. Простейший случай — только преобразование

```python
words = ["python", "java", "go"]
upper = [w.upper() for w in words]
# ['PYTHON', 'JAVA', 'GO']
```

### 2.2. С фильтрацией

```python
numbers = range(20)
divisible_by_3 = [n for n in numbers if n % 3 == 0]
# [0, 3, 6, 9, 12, 15, 18]
```

Важно: `if` в конце (после `for`) — это **фильтр-условие**, а не тернарный оператор. Он решает, попадёт ли элемент в результат, но не позволяет задать альтернативное значение — для этого нужен другой синтаксис (раздел 3).

### 2.3. Несколько условий фильтрации

Можно указывать несколько `if` подряд — они работают как логическое `and`:

```python
result = [n for n in range(50) if n % 2 == 0 if n % 3 == 0]
# эквивалентно: if n % 2 == 0 and n % 3 == 0
# [0, 6, 12, 18, 24, 30, 36, 42, 48]
```

Чаще пишут явным образом через `and`, это читабельнее:
```python
result = [n for n in range(50) if n % 2 == 0 and n % 3 == 0]
```

## 3. Тернарный оператор внутри comprehension (условное выражение)

Чтобы **изменить значение** элемента в зависимости от условия (а не отфильтровать его), тернарный оператор ставится **перед** `for`:

```python
[значение_если_true if условие else значение_если_false for элемент in источник]
```

```python
nums = [-3, -2, -1, 0, 1, 2, 3]
abs_values = [n if n >= 0 else -n for n in nums]
# [3, 2, 1, 0, 1, 2, 3]
```

### Отличие фильтра от тернарного оператора — ключевой момент

```python
# Фильтр: элементы, не прошедшие условие, ПРОПАДАЮТ из результата
only_even = [n for n in range(10) if n % 2 == 0]
# [0, 2, 4, 6, 8]  — длина списка МЕНЬШЕ исходной

# Тернарный оператор: длина результата ВСЕГДА равна длине источника
even_or_odd_label = [n if n % 2 == 0 else -1 for n in range(10)]
# [0, -1, 2, -1, 4, -1, 6, -1, 8, -1]  — длина та же самая
```

### Можно комбинировать оба

```python
result = [n if n % 2 == 0 else -n for n in range(20) if n != 10]
```
Здесь `if n != 10` (в конце) — фильтр, отбрасывает число 10. `n if n % 2 == 0 else -n` (в начале) — меняет значение оставшихся чисел.

## 4. Вложенные циклы в comprehension

Списковое включение может содержать несколько `for` — они работают как вложенные циклы, слева направо (внешний цикл — первый).

```python
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]

# "Разворачиваем" матрицу в плоский список
flat = [num for row in matrix for num in row]
# [1, 2, 3, 4, 5, 6, 7, 8, 9]
```

Это эквивалентно:
```python
flat = []
for row in matrix:
    for num in row:
        flat.append(num)
```

### Декартово произведение (комбинации)

```python
colors = ["red", "green"]
sizes = ["S", "M", "L"]
combinations = [(c, s) for c in colors for s in sizes]
# [('red', 'S'), ('red', 'M'), ('red', 'L'), ('green', 'S'), ('green', 'M'), ('green', 'L')]
```

## 5. Вложенные comprehension (списки списков)

В отличие от нескольких `for` подряд (раздел 4), можно создать **comprehension внутри comprehension** — для обработки матриц с сохранением структуры:

```python
matrix = [[1, 2, 3], [4, 5, 6]]

# Умножаем каждый элемент на 2, сохраняя структуру матрицы
doubled = [[x * 2 for x in row] for row in matrix]
# [[2, 4, 6], [8, 10, 12]]
```

Здесь внешнее `[... for row in matrix]` создаёт список строк, а внутреннее `[x * 2 for x in row]` — обрабатывает каждую строку отдельно, возвращая тоже список. Итог — список списков, то есть матрица той же формы.

**Разница между разделом 4 и разделом 5:**
- Несколько `for` в одном comprehension → **сплющивает** структуру (получаем плоский список).
- Comprehension внутри comprehension (вложенные скобки) → **сохраняет** структуру.

## 6. Dict comprehension

Аналогичная конструкция для словарей, использует фигурные скобки и `ключ: значение`:

```python
{ключ: значение for элемент in источник if условие}
```

```python
words = ["python", "java", "go"]
lengths = {w: len(w) for w in words}
# {'python': 6, 'java': 4, 'go': 2}
```

С условием и тернарным оператором тоже работает:
```python
n_to_parity = {n: ("чёт" if n % 2 == 0 else "нечет") for n in range(5)}
# {0: 'чёт', 1: 'нечет', 2: 'чёт', 3: 'нечет', 4: 'чёт'}
```

Можно строить словарь из двух списков через `zip`:
```python
keys = ["a", "b", "c"]
values = [1, 2, 3]
d = {k: v for k, v in zip(keys, values)}
# {'a': 1, 'b': 2, 'c': 3}
```

## 7. Set comprehension

Тот же синтаксис, что у списка, но в фигурных скобках без `:` — создаёт множество (уникальные элементы, без порядка):

```python
{выражение for элемент in источник if условие}
```

```python
nums = [1, 2, 2, 3, 3, 3, 4]
unique_squares = {n ** 2 for n in nums}
# {16, 1, 4, 9}  (порядок не гарантирован, дубликаты убраны автоматически)
```

## 8. Generator expression — «ленивый» аналог

Синтаксис почти такой же, но в круглых скобках:

```python
gen = (x ** 2 for x in range(1000000))
```

Ключевое отличие от list comprehension:
- **List comprehension** сразу вычисляет **все** элементы и хранит их в памяти как список.
- **Generator expression** ничего не вычисляет заранее — он выдаёт значения по одному, только когда их запрашивают (например, в цикле `for` или через `next()`).

```python
# List comprehension — сразу создаёт список из миллиона чисел в памяти
squares_list = [x**2 for x in range(1_000_000)]

# Generator — почти не занимает память, значения вычисляются "на лету"
squares_gen = (x**2 for x in range(1_000_000))
```

Когда использовать генератор вместо списка:
- Если результат нужен только один раз для перебора (например, передать в `sum()`, `max()`, цикл `for`) и не нужен как список повторно.
- Если данных много и важно экономить память.

```python
total = sum(x**2 for x in range(1_000_000))  # скобки можно даже не дублировать
```

Минус генератора: его можно «пройти» только один раз — после использования он «пустеет».

## 9. Производительность

List comprehension обычно быстрее эквивалентного цикла `for` с `.append()`, потому что:
- цикл добавления реализован на уровне интерпретатора C, а не через явные байт-код операции Python;
- не тратится время на поиск метода `.append` на каждой итерации.

Но разница ощутима только на больших объёмах данных или в горячих участках кода — не стоит жертвовать читаемостью ради минимальной оптимизации там, где это не критично.

## 10. Частые ошибки и антипаттерны

1. **Слишком сложная логика в одну строку.** Если comprehension не помещается на экране и требует более одного уровня вложенности `for`/`if`, лучше вернуться к обычному циклу — код станет читаемее.

2. **Использование comprehension ради побочных эффектов.** Плохо:
   ```python
   [print(x) for x in range(5)]  # так делать не нужно!
   ```
   Comprehension предназначен для создания коллекции, а не для выполнения действий. Здесь просто нужен обычный `for`.

3. **Путаница между фильтром и тернарным оператором** (см. раздел 3) — самая частая ошибка у начинающих.

4. **Затенение переменных.** Переменная цикла в comprehension существует в собственной локальной области видимости (начиная с Python 3) и не «утекает» наружу — это отличается от обычного `for`:
   ```python
   x = 100
   result = [x for x in range(5)]
   print(x)  # всё ещё 100, а не 4
   ```

5. **Чрезмерная вложенность матриц** — тройная вложенность comprehension обычно нечитаема, лучше разбить на явные циклы или вспомогательную функцию.

## 11. Сводная таблица всех видов включений

| Тип | Синтаксис | Результат |
|---|---|---|
| List comprehension | `[x for x in it]` | список |
| Dict comprehension | `{k: v for k, v in it}` | словарь |
| Set comprehension | `{x for x in it}` | множество |
| Generator expression | `(x for x in it)` | генератор (ленивый) |

## 12. Мини-шпаргалка синтаксиса

```python
# Базовое преобразование
[expr for x in iterable]

# С фильтром
[expr for x in iterable if condition]

# С тернарным оператором (условное значение для каждого элемента)
[expr_if_true if condition else expr_if_false for x in iterable]

# С фильтром И тернарным оператором одновременно
[expr_if_true if condition else expr_if_false for x in iterable if filter_condition]

# Вложенные циклы (сплющивание)
[expr for x in outer for y in inner]

# Вложенные comprehension (сохранение структуры)
[[expr for x in row] for row in matrix]
```

1. Создайте список квадратов чисел от 1 до 20.

4. Из строки `"Hello World"` получите список только гласных букв (без учёта регистра).
5. Дан список чисел, получите список их квадратов, но только для чисел больше 5 (фильтр + преобразование одновременно, без тернарного оператора).
6. Из списка целых чисел получите список строк вида `"число: X"`.
7. Дан список слов, получите список их длин.

## С тернарным оператором (9-14)
9. Из списка чисел получите список, где чётные заменены на `"even"`, а нечётные оставлены числами.
10. Из списка чисел от -10 до 10 получите список их модулей без `abs()`.
11. Из списка возрастов получите список меток `"ребёнок"` (<12), `"подросток"` (12-17), `"взрослый"` (18+) — вложенный тернарный оператор внутри comprehension.
12. Из списка слов получите список, где слова длиннее 6 символов обрезаны до 6 символов, остальные — без изменений.
13. Из списка чисел получите список, где отрицательные заменены на 0, а остальные оставлены как есть.
14. Из списка чисел получите список строк "Fizz"/"Buzz"/"FizzBuzz"/само число (тройной вложенный тернарный оператор).

## Вложенные структуры (15-17)
15. Дана матрица 3x3, получите её транспонированную версию с помощью вложенных comprehension.
16. Дана матрица, получите плоский список всех её элементов (сплющивание через двойной `for`).
17. Дан список списков разной длины, получите список сумм каждого подсписка.

## Dict/Set comprehension (18-20)
18. Из списка слов создайте словарь `{слово: длина_слова}`.
19. Из списка чисел создайте множество их остатков от деления на 3.
20. Из двух списков (имена и оценки) создайте словарь `{имя: "сдал"/"не сдал"}` в зависимости от того, оценка >= 60 или нет (используйте `zip` и тернарный оператор в dict comprehension).
