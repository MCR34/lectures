## Часть 1. Общая идея заданий 1–10

Все десять первых заданий устроены одинаково, поэтому сначала разберём **общий метод**, а затем применим его к каждому конкретному заданию — так вы увидите и общую логику, и её частные реализации.

### Что дано

Есть арифметическое выражение вида

```
ЧИСЛО1(в системе счисления p) + ЧИСЛО2(в системе счисления q)
```

В записи обоих чисел на **двух** местах вместо цифр стоят буквы **x** и **y** — это «неизвестные», допустимые цифры соответствующей системы счисления (то есть 0 ≤ x < p или x < q, в зависимости от того, в каком числе стоит цифра, и аналогично для y).

Нужно:
1. подобрать x и y так, чтобы сумма была **кратна** заданному числу **m**;
2. среди всех таких x и y выбрать вариант, где сумма **наименьшая**;
3. в ответе указать частное (сумма ÷ m).

### Решение на Python (общая схема для всех 10 задач)

Все авторские решения на сайте используют одну и ту же конструкцию:

```python
result_search = []
for x in '<допустимые цифры первой системы счисления>':
    for y in '<допустимые цифры второй системы счисления>':
        t = int('<шаблон первого числа с подставленными x,y>', p) + int('<шаблон второго числа>', q)
        if t % m == 0:
            result_search.append(t)
if result_search:
    print(min(result_search) // m)
```

Разберём, что делает каждая строка — это касается **всех** задач 1–10:

1. `result_search = []` — создаём пустой список, куда будем складывать все подходящие (кратные m) суммы.
2. `for x in '0123...':` — перебираем **все возможные цифры x** одну за другой. Строка `'0123456789A...'` — это просто набор символов-цифр нужной системы счисления (для оснований больше 10 после `9` идут буквы `A, B, C...`, как в шестнадцатеричной системе).
3. Вложенный `for y in '...':` — то же самое для y. Двойной цикл перебирает **все пары (x, y)** — это и есть полный перебор (brute force): компьютер честно проверяет вообще все варианты, не пропуская ни одного.
4. `int('88' + x + '4' + y, 9)` — центральная хитрость. Функция `int(строка, основание)` переводит **строку цифр** в число, **интерпретируя её как запись в указанной системе счисления**, и сразу возвращает результат в привычной десятичной системе. Например, `int('105', 8)` вернёт число 105 в восьмеричной системе, переведённое в десятичную (69). Здесь мы просто «собираем» строку из известных цифр и текущих x, y (которые тоже являются символами, так как мы взяли их из строки в цикле `for x in '0123...'`) и сразу получаем десятичное значение всего числа — Python сам разбирается со степенями основания, нам не нужно ничего вычислять вручную.
5. `t = ... + ...` — складываем оба переведённых числа: получаем итоговое значение всего арифметического выражения для конкретных x, y.
6. `if t % 61 == 0:` (число 61/9/56/... своё в каждой задаче) — оператор `%` возвращает **остаток от деления**. Если остаток равен 0 — число делится на m без остатка, то есть кратно ему. Именно так проверяется условие «кратно m» из условия задачи.
7. `result_search.append(t)` — если подошло, добавляем найденное значение в список всех кратных сумм.
8. После двойного цикла: `if result_search: print(min(result_search) // 61)` — если список не пуст, берём **минимальное** значение из всех найденных (`min(...)`) — это и есть наименьшая подходящая сумма, — и делим его на m **целочисленно** (`//`, отбрасывая остаток, хотя остатка и не будет, так как мы уже проверили делимость) — это финальный ответ.

Иногда авторы альтернативных решений на сайте немного меняют форму записи (используют `f-строки` вида `f'88{x}4{y}'` вместо конкатенации через `+`, используют `range(9)` вместо строки цифр и переводят числа в цифры функцией `str()`, или сразу делают `print` и `exit()` внутри цикла, как только нашли первое подходящее значение — раз перебор идёт от меньших x к большим, **первое найденное значение и будет наименьшим**, если x — «главная» переменная и его увеличение гарантированно увеличивает сумму). Суть везде одна и та же.

## Задание 1 (№ 48384)

**Условие (кратко).** Первое число — `88x4y` в системе счисления с основанием 9, второе — `7x44y` в системе счисления с основанием 11. Нужно найти x, y, при которых сумма минимальна и кратна 61, и вычислить частное от деления на 61.

### Решение на Python

```python
result_search = []
for x in '012345678':
    for y in '012345678':
        t = int('88' + x + '4' + y, 9) + int('7' + x + '44' + y, 11)
        if t % 61 == 0:
            result_search.append(t)
if result_search:
    print(min(result_search) // 61)
```

- Строки `'012345678'` — это все 9 цифр девятеричной и одиннадцатеричной систем, которые допустимы для x и y здесь (обратите внимание: и x, и y используются в обоих числах, поэтому диапазон общий — от 0 до 8, так как более узкое ограничение — основание 9).
- `int('88' + x + '4' + y, 9)` собирает строку вида `"88x4y"` (например, при x='3', y='4' получится строка `"8834"`... на самом деле `"8834"` не совсем — там `'88'+'3'+'4'+'4'` = `"88344"`) и переводит её из 9-ричной системы в десятичную.
- Аналогично для второго слагаемого с основанием 11.
- Остальное — по общей схеме, описанной выше: складываем, проверяем остаток от деления на 61, копим подходящие суммы, в конце берём минимум и делим на 61.

**Ответ: 2715.**

## Часть 2. Задания другого типа — 11 и 12

Эти два задания устроены совсем иначе: здесь нет двух чисел в разных системах счисления, а есть **одно большое число**, равное сумме нескольких больших степеней основания, из которого вычитают маленькое число `x`. Нужно понять, сколько цифр, равных `(основание − 1)`, появится в записи результата, и найти наименьший `x`, для которого таких цифр — заданное количество.

## Задание 11 (№ 89197)

**Условие (кратко).** Значение выражения `19²⁷⁰ + 19²⁴⁰ + 19¹⁹⁰ + 19¹⁸⁰ − x` записали в системе счисления с основанием 19. Найти минимальное целое положительное `x`, при котором количество цифр «18» (это и есть `19 − 1`, наибольшая возможная цифра в 19-ричной системе) в записи результата равно 177.

### Решение на Python

```python
for x in range(1, 1000):
    s = 19**270 + 19**240 + 19**190 + 19**180 - x
    t = 0
    while s > 0:
        if s % 19 == 18:
            t += 1
        s //= 19
    if t == 177:
        print(x)
        break
```

Построчно:
1. `for x in range(1, 1000):` — перебираем x от 1 до 999 по порядку возрастания. Так как перебор идёт **от меньшего к большему** и мы прерываем цикл при первом же найденном x (`break`), первое найденное x автоматически будет наименьшим — это и нужно по условию.
2. `s = 19**270 + 19**240 + 19**190 + 19**180 - x` — вычисляем то самое большое число (Python умеет работать с произвольно большими целыми числами без потери точности — это важно, так как `19**270` — число из более чем 300 десятичных цифр).
3. `t = 0` — счётчик цифр, равных 18, обнуляем перед подсчётом.
4. `while s > 0: ... s //= 19` — это стандартный алгоритм **перевода числа в другую систему счисления через последовательное деление на основание**: на каждом шаге берём остаток от деления на 19 (`s % 19`) — это и есть очередная (младшая ещё не обработанная) цифра числа в 19-ричной системе, а затем **отбрасываем** её из числа (`s //= 19` — целочисленное деление, оставляющее старшие разряды). Цикл продолжается, пока не останется `s == 0`, то есть пока не переберём все цифры.
5. `if s % 19 == 18: t += 1` — если очередная цифра равна 18 (то есть максимально возможной цифре при основании 19, `19−1=18`), увеличиваем счётчик.
6. `if t == 177: print(x); break` — как только (после полного перебора всех цифр текущего s) счётчик восемнадцаток равен ровно 177 — выводим найденный x и сразу выходим из внешнего цикла (`break`), так как дальше искать не нужно — это первое (наименьшее) подходящее x.

**Ответ: 382.**

## Задание 12 (№ 89233)

**Условие (кратко).** Значение выражения `14³⁰⁰ + 14²⁴⁰ + 14¹⁷⁰ + 14¹⁵⁰ − x` записали в системе счисления с основанием 14. Найти минимальное `x`, при котором количество цифр «13» (= 14−1) в записи результата равно 148.

### Решение на Python

```python
for x in range(1, 1000):
    s = 14**300 + 14**240 + 14**170 + 14**150 - x
    t = 0
    while s > 0:
        if s % 14 == 13:
            t += 1
        s //= 14
    if t == 148:
        print(x)
        break
```

Код построчно устроен так же, как в задании 11 (замените там 19 на 14, 177 на 148, а степени — на 300/240/170/150):
1. Перебираем x от 1 и по возрастанию — первое найденное значение будет минимальным.
2. Вычисляем большое число `s`.
3. В цикле `while s > 0` переводим `s` в 14-ричную систему счисления «по кусочкам»: `s % 14` даёт очередную младшую цифру, `s //= 14` отбрасывает её и сдвигает к следующей.

## Задание 13 (№ 92191)

**Условие (кратко).** Операнды записаны в системе счисления с основанием 22: `63x89875` + `17x51` + `75x3` (все три числа — 22-ричные, x — одна и та же неизвестная цифра во всех трёх числах). Нужно найти **наибольшее** значение x, при котором сумма кратна 21, и вычислить частное от деления на 21.

Это тот же тип задачи, что и задания 1–10, только здесь неизвестная цифра одна (x), но встречается сразу в трёх числах, и ищется не наименьшая, а **наибольшая** подходящая сумма.

### Решение на Python

```python
digs = '0123456789ABCDEFGHIJKL'  # 22 цифры основания 22: 0-9, затем A-L (10-21)

result_search = []
for i in range(22):
    x = digs[i]
    t = int('63' + x + '89875', 22) + int('17' + x + '51', 22) + int('75' + x + '3', 22)
    if t % 21 == 0:
        result_search.append((i, t))

if result_search:
    best_i, best_t = max(result_search)  # max по первому элементу кортежа — по значению x
    print(best_t // 21)
```

- `digs = '0123456789ABCDEFGHIJKL'` — строка из 22 символов: это все допустимые цифры системы счисления с основанием 22. Первые 10 — обычные цифры 0–9, следующие 12 букв (A, B, C, ..., L) обозначают цифры со значениями 10, 11, ..., 21 (по аналогии с тем, как в шестнадцатеричной системе после 9 идут A–F).
- `for i in range(22):` — перебираем **числовое значение** цифры x от 0 до 21, а `digs[i]` даёт соответствующий ей символ для подстановки в строку числа.
- `int('63' + x + '89875', 22)` — собирает строку `"63x89875"` с текущей подставленной цифрой x и переводит её из 22-ричной системы в десятичную. Аналогично для двух других чисел — `"17x51"` и `"75x3"`.
- `t = ... + ... + ...` — складываем все три числа (в условии их именно три, а не два, как в заданиях 1–10).
- `if t % 21 == 0: result_search.append((i, t))` — если сумма кратна 21, сохраняем **пару** (значение x, сама сумма) в список. Кортеж нужен именно для следующего шага.
- `best_i, best_t = max(result_search)` — здесь ключевое отличие от заданий 1–10: там нужен был **минимум**, а здесь — **максимум**. Функция `max` на списке кортежей сравнивает их поэлементно: сначала по первому элементу (i, то есть по значению x), поэтому автоматически найдётся кортеж с наибольшим x среди всех, что подошли по условию кратности.
- `print(best_t // 21)` — выводим частное от деления найденной суммы на 21.

**Ответ: 733155579.**

*(Для интереса: подходящих значений x всего три — 3, A (10) и H (17); наибольшее из них — H, то есть x = 17.)*

## Задание 14 (№ 92257)

**Условие (кратко).** Значение арифметического выражения

```
5·7776^2013 + 4·1296^2015 − 3·216^2017 + 2·36^2017 − 6^2019 + 2023
```

записали в системе счисления с основанием 36. Нужно определить, сколько цифр в этой записи **не превышают 9** (то есть равны 0, 1, 2, ..., 9 — обычным «арабским» цифрам, а не «буквенным» цифрам от 10 до 35).

Это задание похоже по типу на № 11–12 (одно огромное число, требуется посчитать определённые цифры в его записи в другой системе счисления), но здесь не нужно подбирать x — число уже полностью задано, и достаточно один раз перевести его в 36-ричную систему и посчитать нужные цифры.

### Решение на Python

```python
s = 5 * 7776**2013 + 4 * 1296**2015 - 3 * 216**2017 + 2 * 36**2017 - 6**2019 + 2023

count = 0
while s > 0:
    if s % 36 <= 9:
        count += 1
    s //= 36

print(count)
```

- `s = 5 * 7776**2013 + ... + 2023` — вычисляем значение выражения напрямую по формуле из условия. В Python `**` — это возведение в степень, а целые числа не ограничены по размеру (в отличие от многих других языков), поэтому Python спокойно посчитает число из тысяч десятичных цифр без переполнения.
- `count = 0` — счётчик цифр, значение которых не превышает 9.
- `while s > 0: ... s //= 36` — тот же приём перевода числа в другую систему счисления, что и в заданиях 11–12: на каждом шаге `s % 36` даёт очередную (младшую из ещё не обработанных) цифру числа в 36-ричной записи, а `s //= 36` отбрасывает её и подготавливает следующую.
- `if s % 36 <= 9: count += 1` — если очередная цифра лежит в диапазоне от 0 до 9 (то есть это «обычная» цифра, а не одна из «буквенных» — A, B, ..., Z, обозначающих значения от 10 до 35), увеличиваем счётчик.
- `print(count)` — после того как цикл переберёт все разряды числа (пока не останется `s == 0`), выводим итоговое количество нужных цифр.

**Ответ: 3018.**

# Разбор заданий № 14 ЕГЭ по информатике

Тема: **«Операции в разных системах счисления с двумя переменными»**
Источник: https://inf-ege.sdamgia.ru/test?theme=274

---

## Часть 1. Общая идея заданий 1–10

Все десять первых заданий устроены одинаково, поэтому сначала разберём **общий метод**, а затем применим его к каждому конкретному заданию — так вы увидите и общую логику, и её частные реализации.

### Что дано

Есть арифметическое выражение вида

```
ЧИСЛО1(в системе счисления p) + ЧИСЛО2(в системе счисления q)
```

В записи обоих чисел на **двух** местах вместо цифр стоят буквы **x** и **y** — это «неизвестные», допустимые цифры соответствующей системы счисления (то есть 0 ≤ x < p или x < q, в зависимости от того, в каком числе стоит цифра, и аналогично для y).

Нужно:
1. подобрать x и y так, чтобы сумма была **кратна** заданному числу **m**;
2. среди всех таких x и y выбрать вариант, где сумма **наименьшая**;
3. в ответе указать частное (сумма ÷ m).

### Решение на Python (общая схема для всех 10 задач)

Все авторские решения на сайте используют одну и ту же конструкцию:

```python
result_search = []
for x in '<допустимые цифры первой системы счисления>':
    for y in '<допустимые цифры второй системы счисления>':
        t = int('<шаблон первого числа с подставленными x,y>', p) + int('<шаблон второго числа>', q)
        if t % m == 0:
            result_search.append(t)
if result_search:
    print(min(result_search) // m)
```

Разберём, что делает каждая строка — это касается **всех** задач 1–10:

1. `result_search = []` — создаём пустой список, куда будем складывать все подходящие (кратные m) суммы.
2. `for x in '0123...':` — перебираем **все возможные цифры x** одну за другой. Строка `'0123456789A...'` — это просто набор символов-цифр нужной системы счисления (для оснований больше 10 после `9` идут буквы `A, B, C...`, как в шестнадцатеричной системе).
3. Вложенный `for y in '...':` — то же самое для y. Двойной цикл перебирает **все пары (x, y)** — это и есть полный перебор (brute force): компьютер честно проверяет вообще все варианты, не пропуская ни одного.
4. `int('88' + x + '4' + y, 9)` — центральная хитрость. Функция `int(строка, основание)` переводит **строку цифр** в число, **интерпретируя её как запись в указанной системе счисления**, и сразу возвращает результат в привычной десятичной системе. Например, `int('105', 8)` вернёт число 105 в восьмеричной системе, переведённое в десятичную (69). Здесь мы просто «собираем» строку из известных цифр и текущих x, y (которые тоже являются символами, так как мы взяли их из строки в цикле `for x in '0123...'`) и сразу получаем десятичное значение всего числа — Python сам разбирается со степенями основания, нам не нужно ничего вычислять вручную.
5. `t = ... + ...` — складываем оба переведённых числа: получаем итоговое значение всего арифметического выражения для конкретных x, y.
6. `if t % 61 == 0:` (число 61/9/56/... своё в каждой задаче) — оператор `%` возвращает **остаток от деления**. Если остаток равен 0 — число делится на m без остатка, то есть кратно ему. Именно так проверяется условие «кратно m» из условия задачи.
7. `result_search.append(t)` — если подошло, добавляем найденное значение в список всех кратных сумм.
8. После двойного цикла: `if result_search: print(min(result_search) // 61)` — если список не пуст, берём **минимальное** значение из всех найденных (`min(...)`) — это и есть наименьшая подходящая сумма, — и делим его на m **целочисленно** (`//`, отбрасывая остаток, хотя остатка и не будет, так как мы уже проверили делимость) — это финальный ответ.

Иногда авторы альтернативных решений на сайте немного меняют форму записи (используют `f-строки` вида `f'88{x}4{y}'` вместо конкатенации через `+`, используют `range(9)` вместо строки цифр и переводят числа в цифры функцией `str()`, или сразу делают `print` и `exit()` внутри цикла, как только нашли первое подходящее значение — раз перебор идёт от меньших x к большим, **первое найденное значение и будет наименьшим**, если x — «главная» переменная и его увеличение гарантированно увеличивает сумму). Суть везде одна и та же.

```python
        if t % 89 == 0:

if result_search:

```

## Задание 11 (№ 89197)

**Условие (кратко).** Значение выражения `19²⁷⁰ + 19²⁴⁰ + 19¹⁹⁰ + 19¹⁸⁰ − x` записали в системе счисления с основанием 19. Найти минимальное целое положительное `x`, при котором количество цифр «18» (это и есть `19 − 1`, наибольшая возможная цифра в 19-ричной системе) в записи результата равно 177.

### Решение на Python

```python
for x in range(1, 1000):
    s = 19**270 + 19**240 + 19**190 + 19**180 - x
    t = 0
    while s > 0:
        if s % 19 == 18:
            t += 1
        s //= 19
    if t == 177:
        print(x)
        break
```

Построчно:
1. `for x in range(1, 1000):` — перебираем x от 1 до 999 по порядку возрастания. Так как перебор идёт **от меньшего к большему** и мы прерываем цикл при первом же найденном x (`break`), первое найденное x автоматически будет наименьшим — это и нужно по условию.
2. `s = 19**270 + 19**240 + 19**190 + 19**180 - x` — вычисляем то самое большое число (Python умеет работать с произвольно большими целыми числами без потери точности — это важно, так как `19**270` — число из более чем 300 десятичных цифр).
3. `t = 0` — счётчик цифр, равных 18, обнуляем перед подсчётом.
4. `while s > 0: ... s //= 19` — это стандартный алгоритм **перевода числа в другую систему счисления через последовательное деление на основание**: на каждом шаге берём остаток от деления на 19 (`s % 19`) — это и есть очередная (младшая ещё не обработанная) цифра числа в 19-ричной системе, а затем **отбрасываем** её из числа (`s //= 19` — целочисленное деление, оставляющее старшие разряды). Цикл продолжается, пока не останется `s == 0`, то есть пока не переберём все цифры.
5. `if s % 19 == 18: t += 1` — если очередная цифра равна 18 (то есть максимально возможной цифре при основании 19, `19−1=18`), увеличиваем счётчик.
6. `if t == 177: print(x); break` — как только (после полного перебора всех цифр текущего s) счётчик восемнадцаток равен ровно 177 — выводим найденный x и сразу выходим из внешнего цикла (`break`), так как дальше искать не нужно — это первое (наименьшее) подходящее x.

**Ответ: 382.**
