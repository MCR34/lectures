# Разбор заданий № 14 ЕГЭ по информатике

Тема: **«Операции в разных системах счисления с одной переменной»**
Источник: https://inf-ege.sdamgia.ru/test?theme=239

Всего в теме 11 заданий: 10 «однотипных» заданий (№ 1–10 — два числа в разных системах счисления с одной неизвестной цифрой x) и 1 задание другого типа (№ 11 — одно большое число, из которого вычитают x, нужно посчитать нули в его записи).

---

## Часть 1. Общая идея заданий 1–10

### Что дано

Есть арифметическое выражение вида

```
ЧИСЛО1(в системе счисления p) + ЧИСЛО2(в системе счисления q)
```

В записи обоих чисел на **одном и том же** месте (в каждом из чисел) стоит одна и та же неизвестная цифра **x**. В отличие от «двухпеременных» заданий (где были x и y), здесь неизвестная всего одна, но она может входить в оба числа сразу (на разных позициях в каждом из них).

Важный нюанс — диапазон допустимых значений x бывает двух видов, и это прямо написано в условии:
- если в условии написано «неизвестная цифра **из алфавита десятичной системы счисления**» — значит x может быть только обычной десятичной цифрой: 0, 1, 2, ..., 9, даже если сами числа записаны в системе счисления с намного большим основанием (например, 17 или 18);
- если в условии написано «**допустимые в данных системах счисления** неизвестные цифры» — значит x может принимать любое значение, допустимое по основанию (например, для шестнадцатеричной системы — от 0 до 15, с буквами A–F).

Нужно найти x, при котором сумма кратна заданному числу **m**, причём среди подходящих x берём наименьшее (в отличие от части заданий из предыдущей темы, где было наименьшее *значение суммы* — здесь наименьшим ищут само x). Затем для найденного x нужно вычислить частное от деления суммы на m.

### Решение на Python (общая схема)

Все решения на сайте используют один и тот же приём:

```python
for x in '<допустимые цифры>':
    t = int('<шаблон первого числа с x>', p) + int('<шаблон второго числа с x>', q)
    if t % m == 0:
        print(t // m)
        break
```

Построчный разбор — общий для всех задач 1–10:

1. `for x in '0123456789':` (или другой набор цифр, если по условию x может быть «буквенной» цифрой) — перебираем допустимые значения x **по возрастанию**, один символ за другим.
2. `int('4C' + x + '4', 15)` — собираем строку с текущим x, подставленным на нужное место внутри числа, и переводим её из указанной системы счисления в десятичную. Так же поступаем со вторым числом (возможно, с другим основанием).
3. `t = ... + ...` — складываем оба переведённых числа.
4. `if t % m == 0:` — проверяем, кратна ли сумма нужному числу m (остаток от деления `%` равен 0).
5. `print(t // m); break` — если условие выполнено, сразу выводим частное от деления и **прерываем цикл** (`break`). Поскольку x перебирается строго по возрастанию, первое найденное подходящее значение автоматически оказывается наименьшим — дальше искать не нужно, `break` останавливает цикл сразу, экономя время.

Это ключевое отличие от программ из предыдущей темы (где неизвестных было две и там сначала собирали **все** подходящие суммы в список, а потом брали `min` от списка): здесь, поскольку неизвестная всего одна и перебор идёт строго по возрастанию, `break` при первом совпадении сразу даёт нужный (наименьший) ответ, без необходимости хранить и сравнивать несколько кандидатов.

## Задание 1 (№ 48394)

**Условие (кратко).** Числа `4Cx4` (основание 15) и `x62A` (основание 13). x — обычная десятичная цифра (0–9). Найти наименьшее x, при котором сумма кратна 121, и вычислить частное.

### Решение на Python

```python
for x in '0123456789':
    t = int('4C' + x + '4', 15) + int('' + x + '62A', 13)
    if t % 121 == 0:
        print(t // 121)
        break
```

- `'0123456789'` — x здесь именно десятичная цифра, как и указано в условии, а не «шире», несмотря на то что основания чисел (15 и 13) допускали бы и большие значения цифры.
- `int('4C' + x + '4', 15)` — собирает строку `"4Cx4"` (буква `C` — это фиксированная цифра со значением 12, уже данная в условии, а не переменная) и переводит из 15-ричной системы.
- `int('' + x + '62A', 13)` — аналогично собирает `"x62A"` и переводит из 13-ричной системы (пустая строка `''` в начале — просто стилистическая деталь оригинального кода, не влияет на результат).
- Дальше — сложение, проверка кратности 121 и вывод частного при первом же подходящем x.

**Ответ: 234** (при x = 5).

**Условие (кратко).** Числа `2x84` (основание 19) и `2B3x` (основание 16). Здесь x — **любая допустимая цифра** соответствующих систем счисления (не только десятичная!). Кратность — 88.

## Задание 11 (№ 92258)

**Условие (кратко).** Значение выражения `5^150 + 5^100 − x`, где x — целое положительное число, меньшее 2030, записали в пятеричной системе счисления. Нужно найти наименьшее x, при котором количество **нулей** в пятеричной записи этого числа максимально.

Это задание другого типа: как и в заданиях про «18» и «13» из предыдущей темы, здесь используется эффект заимствования при вычитании небольшого числа из числа с длинной цепочкой нулей в старших разрядах, только теперь считаем не цифры вида «основание минус один», а именно **нули**, и вдобавок ищем не конкретное количество, а **максимум** — то есть нужно перебрать все x из заданного диапазона и найти тот, что даёт больше всего нулей.

### Решение на Python

```python
maxi = 0
otv = 0
for x in range(2030, 1, -1):
    t = 5**150 + 5**100 - x
    c = 0
    while t != 0:
        if t % 5 == 0:
            c += 1
        t //= 5
    if c >= maxi:
        otv = x
        maxi = max(c, maxi)

print(otv)
```

- `maxi = 0` и `otv = 0` — заводим две переменные: `maxi` хранит наибольшее найденное количество нулей, а `otv` — то значение x, при котором это количество было достигнуто (это и есть ответ).
- `for x in range(2030, 1, -1):` — перебираем x от 2029 **вниз** до 2 (третий параметр `-1` означает шаг «минус один», то есть убывание). Это важно: перебор идёт от **больших** значений x к **малым**.
- `t = 5**150 + 5**100 - x` — вычисляем само число (Python поддерживает сколь угодно большие целые числа, поэтому такие огромные степени считаются без проблем).
- `c = 0` — счётчик нулей для текущего x, обнуляем перед подсчётом.
- `while t != 0: ... t //= 5` — стандартный перевод числа в пятеричную систему счисления через последовательное деление на 5: `t % 5` даёт очередную (ещё не обработанную младшую) цифру, `t //= 5` отбрасывает её.
- `if t % 5 == 0: c += 1` — если очередная цифра равна 0, увеличиваем счётчик нулей.
- `if c >= maxi: otv = x; maxi = max(c, maxi)` — вот здесь и происходит основной трюк. Если у текущего x количество нулей `c` **больше или равно** текущему рекорду `maxi`, мы **обновляем** `otv` на это x и подтягиваем `maxi` до `c` (если оно оказалось больше). Обратите внимание на `>=`, а не строгое `>` — из-за этого при **равном** количестве нулей `otv` всё равно перезаписывается новым (более поздним по порядку перебора, то есть **меньшим**, так как перебор идёт по убыванию) значением x. Из-за этого к концу цикла в `otv` останется **самое маленькое** x среди всех, что дали максимальное количество нулей — а это как раз и есть ответ на вопрос «наименьшее x, при котором количество нулей максимально».
- `print(otv)` — выводим найденный ответ.

**Ответ: 625.**

## Главные выводы для самостоятельного решения похожих задач

1. **Для заданий типа 1–10** (два числа с одной общей неизвестной цифрой x): решение — одиночный перебор всех допустимых значений x по возрастанию, перевод обоих чисел в десятичную систему через `int(строка, основание)`, проверка остатка от деления (`% m == 0`) и немедленный вывод результата с `break` при первом совпадении — благодаря возрастающему перебору первое найденное x автоматически оказывается наименьшим.
2. Внимательно читайте условие: если там сказано «цифра из алфавита **десятичной** системы счисления» — x ограничен цифрами 0–9, даже если основания чисел больше; если сказано «допустимые **в данных системах счисления**» — берите полный алфавит по более строгому (меньшему) из двух оснований.
3. Если x стоит на **первом (старшем)** месте числа, перебор логично начинать с 1, а не с 0 (задание 10), хотя формально код может работать и без этого ограничения (задание 5).
4. **Для задания типа 11** (одно большое число минус x, нужно найти экстремум количества определённых цифр): перебор x идёт по всему заданному диапазону (здесь — по убыванию), для каждого x число переводится в нужную систему счисления через `while t != 0: ... t //= b`, подсчитываются нужные цифры, а условие обновления ответа (`>=` вместо `>` при переборе по убыванию) позволяет одновременно найти и максимум count, и наименьшее x, дающее этот максимум, — без необходимости хранить и сравнивать весь список кандидатов.
